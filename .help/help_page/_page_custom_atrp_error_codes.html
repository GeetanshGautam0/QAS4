QUIZZING APP ERROR CLASSES

aTRP codes specify the specific class of error raised. Custom exceptions defined by the quizzing app also appear here.

Non-Default Exception Types

NotYetImplemented Error

  • Defined in qa_std/qa_def.py
  • aTRP: 0x6d155deb1f29ceb8857ef9e4017adf55

The requested feature has not been programmed into the application yet. If you are using:

  1. A BETA or ALPHA version of the app: This is not a bug; this is the result of the app being in its development phase and the feature will likely be programmed into the app in subsequent updates.
  2. A STABLE RELEASE of the app: Report this as a bug at the bug report URL.

BadFileFormat Error

  • Defined in qa_file_io/qa_file_std.py
  • aTRP: 0x6d155deb1f29ceb8857ef9e4017adf55

The Quizzing Application FileIO manager tried to read from a file that was not of the expected format; the erroneous formatting may be part of the header and/or body sections of the file.

FailureToComplyWithPolicy Error

  • Defined in qa_std/qa_def.py
  • aTRP: 0x6d155deb1f29ceb8857ef9e4017adf55

The application did not comply with the policies put forth by the app developer. The policy violated is specified in the error string.
The following are the policies currently defined:

  • POLICY_MAX_IO_EVENTS_PER_MINUTE (default: 15 000 IO events per minute)
  • POLICY_FIO_SW_COMPRESS_SRC_FILE (default: TRUE)
  • POLICY_SW_GZIP_COMPRESSION_LEVEL (default: 7)
  • POLICY_MAX_WHILE_LOOP_RETRIES (default: 100 000 retries)
  • POLICY_FO_GEN_THEME_FILE_VERSION (default: version 1)
